# ЗАДАЧИ (КЛИЕНТ)

> Перед возвратом к серверным задачам проговори, что клиентские пункты завершены/заморожены, чтобы команда могла безопасно переключиться.

## 1. Подключение

* [ ] WebSocket-клиент
* [ ] Автоматический reconnect
* [ ] Отправка pong на ping

## 2. Обработка server_command

* [ ] Сделать клиентский dispatcher для `server_command`
* [ ] По `command` запускать нужное действие
* [ ] Сразу отправлять:
  * `command_received` (опционально)
* [ ] После выполнения:
  * `command_result`

## 3. Выполнение действий (agent logic)

* [ ] Реализовать отдельные методы для команд:
  * Move
  * Scan
  * Stop
  * Idle / Reset
* [ ] Логировать выполнение

## 4. agent_status (обязательное)

* [ ] Запускать таймер: отправлять статус каждые N секунд
* [ ] В payload передавать:
  * режим
  * состояние
  * текущую задачу
  * ошибки
  * позицию (если есть)
* [ ] Ставить timestamp

## 5. UI (опционально)

* [ ] CLI-логирование входящих сообщений
* [ ] GUI (если нужен — Flet)

## Итог

Клиент будет:

* поддерживать стабильное соединение;
* принимать и исполнять команды;
* отправлять подтверждения и результаты;
* регулярно публиковать `agent_status` и вести журнал действий.
